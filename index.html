Clarity: AI Assistant Platform with Voice and Vision

  • Fully functional robot assistant with real-time speech interaction
  • Edge-based facial recognition with dynamic servo control
  • Natural language understanding via OpenAI API
▼

Clarity Robot - GitHub Repo

Clarity is a sophisticated autonomous robot system, featuring a quadruped design with an OLED display and advanced AI capabilities. The robot combines computer vision, natural language processing, and real-time communication to create an interactive companion that can track faces, respond to voice commands, and display emotions through its screen interface. The system architecture utilizes a hybrid onboard/distributed processing approach, Connecting to a powerful core server for video and speech processing when available, then seamlessly falling back to onboard processing when connection to the core server is unavailable.

The robot's personality is driven by OpenAI API integration, allowing for natural conversation. Clarity processes speech input, can perform web searches for real-time information, and responds with synthesized speech while displaying appropriate facial expressions. The face tracking system uses MTCNN neural networks for real-time detection, enabling the robot to maintain eye contact during conversations. Technical highlights include custom servo control for head movement, I2C-based OLED display management, multi-process architecture for concurrent audio/visual processing, and robust network communication protocols for seamless integration between the robot hardware and the core server.

Beyond its technical capabilities, Clarity's system prompt adds multiple fun quirks and easter eggs, adding an element of unpredictability to interactions. The project demonstrates proficiency in embedded systems programming, computer vision, natural language processing, robotics control, and distributed system design, showcasing the integration of multiple complex technologies into a cohesive and engaging robotic platform.

Friend Signals IoT Communication System

IoT Communication Network - GitHub Repo

I developed a distributed IoT alert system that enables remote activation of physical alarm devices through Discord commands. The system consists of three main components: a Python program running on a cloud server that manages connected devices, MicroPython-based client firmware for Raspberry Pi Pico microcontrollers, and a Discord bot interface that allows users to trigger alerts remotely.

The networking protocol is completely custom and built using sockets. Each client device registers with a unique identifier during its handshake protocol, and the server continuously monitors device health through periodic keep-alive messages, automatically pruning disconnected devices from the active client list.

The client firmware runs on embedded devices equipped with LEDs, buzzers, and WiFi connectivity. When triggered, devices execute a coordinated audio/visual alert sequence featuring alternating LED patterns and PWM-controlled buzzer tones, creating an effective alert that your friends are interested in speaking to you, at any hour of the night! The firmware includes robust error handling with watchdog timer integration to ensure reliable operation and automatic recovery from network failures or system crashes.

This project demonstrates proficiency in network programming, embedded systems development, asynchronous programming patterns, and IoT device management. The architecture emphasizes reliability and scalability, with proper connection management, error handling, and real-time device monitoring capabilities that make it suitable for production deployment in environments requiring reliable remote notification systems. It is my most technically sound (and gloriously annoying) project.

Smart Plant Monitoring System

I developed a Bluetooth-enabled smart plant monitoring system that provides real-time soil moisture readings through both a local OLED display and wireless connectivity.

The system is built around an Arduino Nano ESP32, equipped with a capacitive soil moisture sensor and a 128x64 pixel OLED display. The moisture sensor uses analog-to-digital conversion to measure soil conductivity, which is then calibrated and mapped to percentage values between completely dry and fully saturated conditions. The firmware implements custom calibration constants.

The user interface features an OLED display that presents multiple data visualizations simultaneously. The screen shows raw sensor values, percentage-based moisture levels for easy interpretation, descriptive status text ranging from "Very Dry" to "Very Wet," and a dynamic progress bar that provides immediate visual feedback. In addition to visual output, the device acts as a Bluetooth peripheral, allowing the Clarity robot to interface with it and poll the sensor data on request.

This project demonstrates expertise in embedded C++ programming, sensor interfacing, display graphics programming, Bluetooth Low Energy protocols, and IoT device development.

Environmental Monitoring Station

I developed a multi-sensor telemetry system using the Zephyr Real-Time Operating System that aggregates environmental data from multiple sensors and provides wireless access to the Clarity robot through Bluetooth Low Energy. This embedded system demonstrates advanced RTOS programming, sensor integration, and wireless communication protocols in a production-ready IoT device architecture.

The system integrates a VL53L0X time-of-flight laser distance sensor for precise ranging measurements, and an HTS221 environmental sensor providing both temperature and humidity readings. The firmware implements robust sensor initialization with device tree integration, ensuring proper hardware abstraction and driver compatibility within the Zephyr ecosystem. Each sensor is polled independently with error handling.

Clarity's system prompt includes a tool call to connect to the station and poll the data, allowing the robot to determine the ambient temperature or humidity when asked. This project showcases expertise in RTOS development, embedded systems architecture, sensor interfacing protocols, Bluetooth Low Energy specification implementation, and real-time data processing.
